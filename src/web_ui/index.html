Circuits

Cap is enforced from the live plug measurements (signed sum). If any plug in a circuit goes stale > plug_stale_s, the entire circuit is muted: CT silent for group_silent_after_stale_s so every Marstek's watchdog clears its integrator. Resumes once plugs are healthy again.

Batteries

Each battery has its own virtual integrator (commanded), the cumulative sum of every delta we have pulsed to its Marstek. The Shelly Plug PM Gen3 reading (plug) is the ground truth: drift between commanded and plug for > saturation_window_s triggers saturation, which caps the battery at the observed ceiling and redispatches the slack to siblings.

Configuration

Pulse-mode config is file-based (read on start). Edit /config/config.toml via the Studio Code Server add-on, then restart this add-on to apply. Live editing in the UI was removed with the multiplex layer — the new architecture depends on a static plug/circuit topology.
